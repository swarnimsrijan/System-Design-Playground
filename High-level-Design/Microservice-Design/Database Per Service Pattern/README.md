# Database Per Service Pattern
- promotes decentralized data mgmt
- assigns unique db to each service
- enhances data encapsulation, autonomy, and scalability
- services to evolve independently without impacting others
- diverse data storage technologies tailored to specific service needs, improving performance and resilience

# Objectives
- Independence
- Encapsulation
- Technology Choice
- Scalability and Performance
- Resilience and Fault Isolation
